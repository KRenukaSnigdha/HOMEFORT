How It Works

HomeFort AI follows a structured IDS pipeline: Capture → Analyze → Store → Alert → Mitigate.

1. Packet Capture

Captures real-time packets from Wi-Fi/Ethernet using Scapy Sniffer and extracts traffic features.

2. ML + Threat Intelligence

Classifies traffic using ML model and validates suspicious IPs using reputation-based scoring.

3. MongoDB Storage & Alerts

Stores packets & alerts securely in MongoDB Atlas, enabling dashboard monitoring per user/device.

MongoDB Atlas Architecture

HomeFort AI supports scalable deployment by storing packets and alerts in MongoDB Atlas. This ensures multi-user access, device-level separation, and research-friendly cloud logging.

User-Based Data Separation

After login, each user is assigned a unique user_id. All stored packets and alerts contain this user_id so one user cannot access another user's traffic logs.

Device-Based Logging

Each monitored device is assigned a device_id. MongoDB stores packets separately for each device, allowing multi-device home network security monitoring.

Cloud Storage for Deployment

MongoDB Atlas provides a managed cloud database. This is required for production deployment and allows research-grade scalability for IEEE paper results.

Collections Used
